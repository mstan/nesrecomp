# Phase 1 — interpreter fallback: the boundary-handoff contract

Branch `explore/multitier-recomp` (worktree `_wt-multitier`). This is the design to lock
*before* writing the interpreter. Grounded in the actual codegen
(`recompiler/src/code_generator.c`) and runtime (`runner/src/runtime.c`).

## 0. What we learned from the code (the constraints that shape everything)

1. **Two stacks, kept in sync by `push_all_jsr`.** A native JSR (`code_generator.c:1455`)
   pushes the real `PC+2` onto the 6502 RAM stack (`g_ram[0x100+S]`, `g_cpu.S--` twice)
   *and* makes a C call. A native RTS (`:1830`) does `g_cpu.S += 2`. So **the 6502 stack
   depth mirrors the C call depth** — this is the property the whole contract leans on.
2. **The miss seam already balances the stack.** A dynamic JSR emits
   `if (!call_by_address(0xADDR)) g_cpu.S += 2;` (`:1498`). On a miss it pops the pushed
   return so the stack stays balanced and the routine simply didn't run. Tail transfers
   (JMP-indirect `:1743`, RTS-as-computed-goto `:1824`, cross-fn branch `:1210`) emit
   `call_by_address(addr); return;` with the *dispatched routine* expected to pop the
   caller's return ("the dispatched function's RTS pops the caller's JSR return … don't
   double-pop", `:1826`).
3. **The depth-510 trap (the hard constraint).** `code_generator.c:2442-2452`: a fallback
   that re-dispatched a *mid-function goto target* through `call_by_address` "re-enters the
   world from scratch and grows the C stack indefinitely … the chain overflowed at depth
   510." **Therefore the interpreter must NEVER `call_by_address` a return address to
   resume native code, and must NOT use C recursion for nested missed calls.**
4. **Universal S-bail.** After a pushing JSR, codegen emits
   `if (g_cpu.S != _cbs) return;` (`:1573`) to propagate multi-level stack unwinds
   (PLA/PLA/RTS bail). The interpreter must leave `g_cpu.S` in the state native code
   expects so this check stays correct.
5. **Coroutines are setjmp/longjmp fibers** (`coroutine.h`). The scheduler dispatches via
   `call_by_address` (START) or `coroutine_resume` (longjmp). A yield longjmps out of the
   C frame. The interpreter cannot model this — it must hand off to native for coroutine
   primitives (they're recompiled), and a longjmp through an interpreted frame is a known
   edge case (see §6).
6. **2A03 has no decimal mode** — ADC/SBC ignore the D flag, so the interpreter skips BCD
   (matches the recompiler and real NES). One fewer correctness pit.

## 1. Integration point — inside `call_by_address`'s miss path (not the call sites)

The interpreter hooks in exactly one place: the **miss paths of `call_by_address`** — the
`default:` case, the inner `switch(g_current_bank) default:`, and the `addr < 0x8000`
guard, all generated by `emit_dispatch` (`code_generator.c:2482`). Today they emit
`nes_log_dispatch_miss(addr); return 0;`. They will instead emit:

```c
return nes_interp_dispatch(addr);   /* logs the miss, runs the interpreter, returns 1 if handled */
```

This is a **codegen change** (`emit_dispatch` in `code_generator.c`) + a new **runtime**
function — never a hand-edit of generated files (RULE 1). Hooking *inside*
`call_by_address` (rather than at each call site) is what makes it uniform: every call
site already calls `call_by_address`, and the seam's `if (!call_by_address(..)) S += 2`
and tail `call_by_address(..); return;` forms both behave correctly when the interpreter
stands in for a successful dispatch and returns 1 (see §2 — the S contract makes both
cases collapse to the same rule).

`nes_interp_dispatch(addr)`:
1. honor `game_dispatch_override(addr)` first (preserves Zelda's SRAM remap until it's
   subsumed),
2. record the miss via the existing ring/log machinery (refactor `nes_log_dispatch_miss`
   so the *recording* half is reusable as `nes_record_dispatch_miss`) — so the discovery
   data is still captured for the Phase 2 manifest even though the game keeps running,
3. run `interp_run(addr)` under the watchdog,
4. return 1 if handled, 0 to fall back to today's behavior (so a tripped watchdog or a
   disabled-by-env interpreter degrades to the current `return 0`).

## 2. The boundary-handoff contract (the core)

`int interp_run(uint16_t addr)` stands in for a successful `call_by_address(addr)`. It runs
its **own** program counter `ipc` (never `call_by_address` on a return address). On entry:

```
S_floor = g_cpu.S      // the 6502 stack level of the routine's own frame
ipc     = addr
```

Loop, one guest instruction per iteration (fetch via §3, execute against `g_cpu` + bus):

- **ALU / load / store / register / flag ops** — execute, advance `ipc`. Stores go through
  the real bus (`nes_write`) so mapper bank switches and PPU/APU writes take effect.
- **Local branch (Bcc) / JMP-absolute within interpreted code** — set `ipc` to target.
- **JSR `target`**:
  - push `ipc+2` to the 6502 stack (6502 semantics; `g_cpu.S -= 2`),
  - if `target` is **covered** by native dispatch → `call_by_address(target)` (native runs,
    its RTS pops our push, returns), then continue at `ipc+3`;
  - else (`target` also un-recompiled) → just set `ipc = target`. **Nested missed calls stay
    inside this one C frame**, tracked on the 6502 stack — never C recursion (§0.3).
- **JMP-indirect / RTS-as-computed-goto `target`** (tail):
  - if covered → `call_by_address(target)`; else `ipc = target`.
- **RTS**: pop 2 bytes → `ret`; `g_cpu.S += 2`. This is the crux:
  - if `g_cpu.S > S_floor` → the routine has returned **to its caller** (native C, up the
    stack). **Return 1 from `interp_run`. Do NOT dispatch `ret`.** (This is the depth-510
    lesson: the C stack / continuation carries the return.)
  - else (still inside a nested interpreted call) → `ipc = ret + 1`, continue.
- **After every step, if `g_cpu.S > S_floor` → return 1.** This single rule also terminates
  the tail-JMP-into-native case (the native target's RTS pops past `S_floor`).
- **RTI**: restore P + pop return like the codegen does (`:1836`); treat the return like
  RTS for the `S_floor` test.
- **BRK / unknown / illegal opcode** — record + bail (return 0) unless the illegal is
  implemented (add on demand).

**Why this is correct for both miss flavors** (same rule, no special-casing):
- *Dynamic JSR miss* (`if(!call_by_address(addr)) S+=2`): the JSR already pushed the return,
  so `S_floor` is the post-push level; the routine's terminating RTS lifts `S` above
  `S_floor`, we return 1, and the seam **skips** its `S += 2`. Net `S` == the pre-call
  `_cbs` — exactly the native-callee post-condition, so the §0.4 bail check stays valid.
- *Tail transfer* (`call_by_address(addr); return`): no new push; the routine's RTS pops the
  *current function's* caller-return (lifting `S` above `S_floor`), we return 1, native does
  its `return;`. The caller-return was consumed by the routine — matching the existing
  "dispatched function's RTS pops the caller's JSR return; don't double-pop" convention
  (`:1826`).

## 3. Instruction fetch & bus (match the recompiler, no fetch side-effects)

- **$8000–$FFFF**: `mapper_peek_prg(pc)` — bank-correct, side-effect-free (already used by
  the miss logger, `runtime.c:1102`).
- **$6000–$7FFF**: `g_sram[...]` (this is what makes SRAM-resident code Just Work, §5).
- **$0000–$1FFF**: `g_ram[...]` (RAM-resident / self-built code).
- Operand **data** reads/writes use the same `nes_read`/`nes_write` helpers the recompiled
  code uses (so MMIO side-effects, mirroring, and mapper writes are identical). Code
  *fetch* must not go through `nes_read` (avoid clearing $2002 etc. by fetching).
- **Timing**: call `maybe_trigger_vblank(n)` periodically (e.g. on backward branches and
  every K instructions), mirroring how recompiled code yields to the NMI driver — so an
  interpreted poll-loop on a vblank flag still makes progress.

## 4. Precondition: `push_all_jsr`

The `S_floor` contract requires the 6502 stack to faithfully mirror calls — i.e. the game
must be built with `push_all_jsr = true` (or `push_jsr` covering the relevant chains).
Metroid and Megaman3 already use it; pick a `push_all_jsr` game for the prototype. Without
it, native JSRs don't push to the 6502 stack and RTS-termination detection is unsound — so
`nes_interp_dispatch` should **assert/disable** itself when `push_all_jsr` is off and fall
back to `return 0`. (Generalizing to non-push_all_jsr is out of Phase 1 scope.)

## 5. What this fixes for free

- **Missed dynamic dispatch into ROM** — the routine runs correctly, control returns, the
  game keeps playing; every miss still logged → discover the whole batch in one run.
- **SRAM-resident code (Zelda)** — `addr < 0x8000` now interprets `g_sram` bytes; the
  per-game `sram_map` + `dispatch_override` hack becomes unnecessary (keep honoring the
  override during transition, then retire it).
- **Truly dynamic RAM code** — interprets `g_ram` bytes; impossible today, no other fix.

## 6. Risks / known limitations (Phase 1)

- **Coroutine longjmp through an interpreted frame.** If interpreted code reaches a
  coroutine yield (recompiled → via `call_by_address` handoff) that longjmps out, it skips
  `interp_run`'s return. Missed *leaf* routines rarely yield; detect yield-target handoff
  and document. Revisit only if a real game trips it.
- **Mid-routine NMI preemption.** NMI is driven between frames by the runner, not
  preemptively; `maybe_trigger_vblank` covers the common vblank-poll pattern. A routine
  that depends on true mid-instruction NMI is an edge case.
- **Illegal opcodes** — implement the official 151 first; add illegals on demand (some NES
  games use a few). Unknown opcode → record + bail (return 0), never silently mis-execute.
- **Watchdog** — cap interpreted instructions per `nes_interp_dispatch` (e.g. 100k) and a
  cross-handoff depth guard; on trip, log + bail to TRAP/`return 0` (catches "jumped into
  data"). Bounds runaway garbage without hanging.

## 7. Deliverables & test plan

Files (all in `runner`/`recompiler` — never generated):
- `runner/src/interp.c` + `runner/include/interp.h` — the 6502 interpreter, `interp_run`,
  `nes_interp_dispatch`, watchdog. (~1.5–2.5k LoC.)
- `runner/src/runtime.c` — split `nes_log_dispatch_miss` → reusable
  `nes_record_dispatch_miss`; add env toggle `NESRECOMP_INTERP_FALLBACK` + watchdog cap +
  a per-frame interpreted-instruction counter (the §6.3 measurement hook for the sljit
  decision).
- `recompiler/src/code_generator.c` — `emit_dispatch` miss paths emit
  `return nes_interp_dispatch(addr);`.
- `runner/runner.cmake` — add `interp.c`.

Test:
1. Pick a `push_all_jsr` game with known misses (Megaman3 or Faxanadu). Regenerate + build.
2. Differentially validate `interp_run` against the **Nestopia oracle** (already wired via
   `emu_step`/TCP): from a captured pre-miss state, run the routine in both, compare
   `g_cpu` + RAM.
3. Play through; confirm the game no longer truncates at the first miss, the whole miss set
   shows up in `dispatch_misses.log`, and screenshots match the oracle.
4. Only then generalize rollout and start Phase 2 (manifest JSON + fold tool).
